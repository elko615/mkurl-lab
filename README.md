# mkurl-lab

## 项目简介

mkurl-lab 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，而是提供一个通用的、结构化的空间，用于存放各类 HTML 实验页面、工具页面或静态内容页面。所有页面均以独立文件形式存在，便于浏览、调试和分享。

## 目录说明

仓库根目录下主要包含以下内容：

- `index.html` — 仓库入口页面，通常包含页面列表或导航信息
- `pages/` — 存放所有独立 HTML 页面文件的目录
- `assets/` — 存放页面所需的公共资源（如 CSS、JavaScript、图片等）
- `README.md` — 本文件

每个 HTML 页面应尽量保持独立，避免对外部资源的强依赖。如需共享资源，请统一放置在 `assets/` 目录下。

## 页面归档说明

本仓库中的 HTML 页面可以包括但不限于：

- 工具类页面（如计算器、转换器、生成器等）
- 实验性页面（如前端技术演示、交互原型等）
- 信息展示页面（如数据可视化、文档页等）
- 其他任何适合以独立 HTML 文件形式存在的静态内容

所有页面均以 `.html` 后缀命名，并放置在 `pages/` 目录中。建议在页面内添加简单的元数据注释（如创建日期、作者、用途等），方便后续维护。

## 维护说明

- 新增页面：将 HTML 文件放入 `pages/` 目录，并更新 `index.html` 中的导航链接（如有）。
- 更新页面：直接修改对应 HTML 文件，注意保持文件格式和资源路径正确。
- 删除页面：从 `pages/` 目录中移除文件，并清理 `index.html` 中的相关链接。
- 资源管理：公共资源放入 `assets/` 目录，避免在页面中嵌入过多内联资源。

本仓库采用静态文件结构，不涉及后端服务或动态生成。所有页面均可通过 GitHub Pages 或其他静态托管服务直接访问。

## 许可

本仓库内容仅供学习、实验和归档使用。除非页面内另有说明，否则不提供任何明示或暗示的保证。请尊重他人劳动成果，合理使用。
